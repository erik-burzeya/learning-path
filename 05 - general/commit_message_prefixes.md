# Commit Message Prefixes

## Add

Neue Funktionalität hinzufügen.

**Beispiele**

* Add: PDF export functionality
* Add: Medication reminder system

---

## Fix

Fehler beheben.

**Beispiele**

* Fix: Tab bar icon alignment
* Fix: Export crash on Android

---

## Refactor

Code verbessern oder umstrukturieren, ohne das Verhalten zu ändern.

**Beispiele**

* Refactor: Simplify storage logic
* Refactor: Clean up navigation structure

---

## Update

Bestehende Funktion erweitern oder verbessern.

**Beispiele**

* Update: Improve export screen UX
* Update: Enhance medication form validation

---

## Style

Änderungen am Design, Layout oder Erscheinungsbild.

**Beispiele**

* Style: Adjust dark mode colors
* Style: Improve tab bar spacing

---

## Config

Konfigurationen ändern.

**Beispiele**

* Config: Configure EAS Update
* Config: Update Expo settings

---

## Remove

Code, Dateien oder Funktionen entfernen.

**Beispiele**

* Remove: Unused components
* Remove: Legacy export logic

---

## Docs

Dokumentation erstellen oder aktualisieren.

**Beispiele**

* Docs: Add installation guide
* Docs: Update README

---

## Test

Tests hinzufügen oder anpassen.

**Beispiele**

* Test: Add export feature tests
* Test: Cover medication validation

---

## Build

Build-, Deployment- oder Dependency-bezogene Änderungen.

**Beispiele**

* Build: Update Expo SDK
* Build: Optimize Android build process

---

## Perf

Performance-Verbesserungen.

**Beispiele**

* Perf: Reduce startup time
* Perf: Optimize list rendering

---

## Chore

Allgemeine Wartungsarbeiten, die keiner anderen Kategorie zugeordnet werden können.

**Beispiele**

* Chore: Update dependencies
* Chore: Clean project structure
